# 🚶 TestWalker

Vanilla JS + CSS で動く、手動テストの **手順表示 + 結果記録** ツール。
テスト手順とテストデータの `.json` を渡すと、実施・記録用のUI画面を生成します。
ビルド不要・オフライン動作で、`testwalker.html` をブラウザで開くだけで使えます。

## 特徴
- **環境構築不要** — `testwalker.html` をブラウザで開くだけ。サーバー・ビルド不要、オフライン（`file://`）動作
- **2ファイル入力** — テスト手順JSON（必須）とテストデータJSON（任意）を分離。手順は固定でデータだけ差し替え可能
- **データ駆動** — 1つの手順に複数データセットを与えると、組み合わせごとに実施行を自動生成。手順文中の `{{キー}}` をデータで置換
- **多言語対応** — UI文言は辞書化（ハードコードなし）。ヘッダーで言語切替、選択は保存。日本語／英語を同梱、言語追加は辞書ファイルを1つ足すだけ
- **入力クリア / 例コピー** — 各入力パネルに「クリア」ボタン、JSONフォーマット例に「コピー」ボタン
- **テストデータを1クリックコピー** — 操作手順中の置換済み値（`{{key}}`）をクリックしてコピー（被テストアプリへ貼り付けやすい）。`file://` でも `execCommand` フォールバックで動作
- **ステップ単位の結果記録** — 各ステップの右に「結果」列（select：未実施／OK／NG・色分け）。状態は保存され、ケース見出しに「OK n・NG m・未 k」の内訳を表示。HTMLレポートにも反映
- **Pass ガード** — 全ステップが OK になるまで Pass は押せない（無効化＋ツールチップで理由提示）。Fail / Blocked / Skip は常に可。Pass 後にいずれかのステップを OK 以外へ変えると自動的に未実施へ戻す
- **自動保存** — 入力中に localStorage へ自動保存。閉じても続きから再開
- **HTMLレポート出力** — 結果を見やすい／印刷可能なHTMLとして書き出し（証跡用）
- **進捗サマリ** — Pass / Fail / Blocked / Skip / 未実施 の件数と達成率を常時表示

## ファイル構成
```
testwalker.html      エントリ（マークアップのみ／CSS・JSを読み込む）
css/style.css        スタイル
js/i18n.js           多言語エンジン（t() / 辞書登録 / data-i18n 適用）
js/app.js            アプリ本体（読み込み・展開・記録・レポート）
i18n/ja.js           日本語辞書
i18n/en.js           英語辞書
```

> 言語辞書は `fetch` ではなく `<script>` で読み込むため、`file://` で直接開いても CORS で失敗しません。

## 言語を追加するには
`i18n/xx.js` を作り、既存辞書と同じキーで `TW_I18N.register("xx", { __name__: "表示名", ... })` を呼び、
`testwalker.html` の `<script src="i18n/xx.js">` を1行追加するだけです。
- 入力は ドラッグ＆ドロップ / ファイル選択 / 貼り付け に対応

## 使い方
1. `testwalker.html` をブラウザで開く
2. テスト手順JSON（と任意でデータJSON）を読み込む
3. 「画面を生成 ▶」をクリック
4. 各ケースを開いて手順を確認 → 判定（Pass/Fail/Blocked/Skip）と実測結果を記録
5. 「📄 HTMLレポート出力」で結果を保存

> はじめての場合は、ローダー画面の「JSONフォーマットの例を見る」→「このサンプルを読み込む」で動作を確認できます。

## JSONフォーマット

### テスト手順JSON（必須）
```json
{
  "meta": { "title": "ログイン機能テスト", "version": "1.0" },
  "testCases": [
    {
      "id": "TC-001",
      "title": "メールアドレスとパスワードでログイン",
      "precondition": "対象ユーザーが登録済みであること",
      "steps": [
        { "no": 1, "action": "ログイン画面を開く", "expected": "入力欄が表示される" },
        { "no": 2, "action": "メール「{{email}}」を入力", "expected": "入力値が反映される" },
        { "no": 3, "action": "パスワード「{{password}}」でログイン", "expected": "{{result}}" }
      ]
    }
  ]
}
```

### テストデータJSON（任意）
テストケースIDで紐付け。1ケースに複数データセットを定義でき、各データセットが1実施行になります。
データセットは **入力データ（`data`）だけ**を持ちます（期待結果は手順側の `expected` に書きます）。

- `data` … 本物の入力値。手順文の `{{キー}}` はここから置換され、**クリックでコピー**できる対象
- `label` … 画面上の識別名（データラベル）

```json
{
  "TC-001": [
    { "label": "正常系",       "data": { "email": "user@example.com", "password": "Pass1234" } },
    { "label": "誤パスワード", "data": { "email": "user@example.com", "password": "wrong" } }
  ]
}
```

#### パターン：1項目＝1ケース・複数ステップ・名前付き data
1項目（例：メール欄）のバリデーションを1ケースにまとめる場合、入力パターンごとに**ステップを分け**、
各ステップで**別名の入力キー**を参照します。期待結果は各ステップに書き、結果は OK/NG で個別記録します。

```json
// 手順側（steps）
{ "no": 1, "action": "メール「{{empty}}」を入力し送信",   "expected": "未入力エラー。コード未送信" }
{ "no": 2, "action": "メール「{{noAt}}」を入力し送信",    "expected": "@なしエラー。コード未送信" }
{ "no": 3, "action": "メール「{{valid}}」を入力し送信",   "expected": "認証コードが送信される" }

// データ側（1データセットに名前付き入力をまとめる）
"TC-003": [
  { "label": "メールパターン",
    "data": { "empty": "", "noAt": "notanemail", "valid": "user@example.com" } }
]
```

## 記録項目（テストケース単位）
- ステータス: Pass / Fail / Blocked / Skip（未選択は「未実施」。同じボタン再クリックで解除）
- 実測結果・備考（自由記述）
- 実施者
- 最終更新日時（自動）

## データの保存について
- 記録はブラウザの **localStorage** に `meta.title` + `meta.version` をキーに保存されます
- 別端末・別ブラウザへ共有する／恒久的に残す場合は **HTMLレポート出力** を利用してください
- 「🗑 記録リセット」で当該テストの記録を消去できます
